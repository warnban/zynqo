# zynqo — нейросети по‑русски

Веб‑приложение с доступом к ИИ‑моделям через единый API [AI Tunnel](https://aitunnel.ru)
(оплата в рублях). Без подписки — у пользователя **баланс**, с которого
списываются деньги за каждую генерацию.

> **Деплой на VPS:** пошаговое руководство — [deploy/GUIDE.ru.md](deploy/GUIDE.ru.md). Краткая справка — [deploy/DEPLOY.ru.md](deploy/DEPLOY.ru.md).

Интерфейс намеренно простой: выбрал инструмент → ввёл текст → **видишь цену** → подтвердил → получил результат.
Светлая и тёмная темы.

## Запуск (полный стек)

Нужно два процесса — бэкенд и фронтенд.

**1. Бэкенд** (Node 22+, использует встроенный `node:sqlite`):

```bash
cd server
copy .env.example .env   # Windows  (cp .env.example .env на macOS/Linux)
npm install
npm run dev              # http://localhost:8787
```

**2. Фронтенд** (в отдельном терминале):

```bash
pnpm install
pnpm dev                # http://localhost:5173  (проксирует /api на :8787)
pnpm build              # продакшн-сборка в dist/
```

> Фронтенд: требуется одобрение нативных build‑скриптов (`esbuild`,
> `@tailwindcss/oxide`) — уже включены в `pnpm-workspace.yaml` → `allowBuilds`.

### Аккаунты

- **Регистрация** по e‑mail + пароль, при регистрации начисляется бонус 30 ₽.
- **Администратор** создаётся при первом старте бэкенда из `server/.env`
  (`ADMIN_EMAIL` / `ADMIN_PASSWORD`, по умолчанию `admin@aneuro.local` / `admin12345`).
  У админа в сайдбаре появляется раздел **«Админ‑панель»**.
- **Вход через VK ID** (OAuth 2.1 + PKCE). Если в `server/.env` не задан `VK_CLIENT_ID` —
  кнопка «Войти через VK» скрыта, остальной вход работает как обычно.

### Настройка входа через VK

1. Зарегистрируйте приложение на [id.vk.com](https://id.vk.com) → «Мои приложения».
2. Скопируйте **ID приложения** (числовой) в `VK_CLIENT_ID`.
3. В настройках приложения в «Доверенные Redirect URI» добавьте адрес из `VK_REDIRECT_URI`
   (по умолчанию `http://localhost:8787/api/auth/vk/callback`). Адрес должен совпадать точно.
4. Перезапустите бэкенд. Поток: фронт → `/api/auth/vk/start` → VK ID → `/api/auth/vk/callback`
   (обмен кода на токен по PKCE + получение профиля) → редирект на фронт с нашим JWT.
   Новому VK‑пользователю начисляется приветственный бонус.

> Реализация: `server/vk.js` (PKCE, обмен кода, профиль), маршруты в `server/index.js`,
> кнопка и обработка возврата токена — на фронте (`AuthScreen`, `AuthProvider`).

### Режим генераций

- Если в `server/.env` задан `AITUNNEL_API_KEY` — генерации **реальные** (через AI Tunnel).
- Если ключа нет — **демо‑режим**: чат и транскрибация возвращают заглушки,
  фото — демо‑картинку. Баланс при этом списывается по тем же правилам (удобно тестировать).

## Что уже сделано

**Фронтенд**
- Экраны входа/регистрации, профиль с выходом.
- Студия: сайдбар по категориям, 7 моделей, светлая/тёмная тема.
- Показ цены **до** генерации + модалка подтверждения списания.
- Чат, генерация фото/видео, транскрибация — подключены к реальному API.
- Модалка пополнения с тарифными пакетами и бонусами.
- **Админ‑панель**: обзор/статистика, пользователи (с начислением баланса), генерации, логи.

**Бэкенд** (`server/`)
- Авторизация (JWT + bcrypt), роли `user` / `admin`.
- Баланс и леджер в SQLite с **атомарными** операциями (холд → списание/возврат).
- Прокси к AI Tunnel: чат (по токенам), фото, видео (async polling), транскрибация (multipart).
- Логирование всех запросов; история генераций.
- Админ‑API: статистика, пользователи, генерации, логи, начисление баланса.

## Новичкам и защита от перерегистрации

**Что даём:** приветственный бонус **30 ₽** (после подтверждения телефона) +
**10 бесплатных сообщений/день** на Gemini Flash Lite (удержание).
На видео бонуса намеренно не хватает — это самая дорогая операция и главная цель фарма.

**Защита от халявы (слои):**
1. **SMS‑верификация телефона** — главный барьер: 1 номер = 1 бонус.
2. **Бонус не выводится и не переводится** — только на генерации. Крупные бонусы — после первого пополнения.
3. **Device fingerprint** (FingerprintJS) — ловит мультиаккаунты с одного устройства.
4. **Блок одноразовых почт, IP датацентров, гео‑рейтлимит** на регистрации.
5. **Привязка хэша банковской карты** при первом пополнении.

## Архитектура бэкенда (план)

Ключ AI Tunnel **нельзя** держать на фронтенде — все вызовы идут через бэкенд.

```
[ React (этот репозиторий) ]
        │  HTTPS, сессия/JWT
        ▼
[ API: Node + Fastify ]
   ├─ Auth (телефон + SMS), сессии
   ├─ Леджер баланса (PostgreSQL): атомарное списание + идемпотентность
   ├─ Антифрод-модуль (fingerprint, IP, лимиты)
   ├─ Прокси к AI Tunnel  → https://api.aitunnel.ru/v1  (OpenAI-совместимый)
   ├─ Очередь видео (Kling/Veo: POST /v1/videos → polling) [BullMQ + Redis]
   └─ Платежи (вебхуки ЮKassa / Robokassa / CryptoCloud)
        │
        ▼
[ PostgreSQL ]  users, balances, ledger, generations, payments
[ S3-совместимое хранилище ]  готовые фото/видео
```

### Поток списания (надёжно)

1. Фронт запрашивает оценку стоимости → бэк считает по `catalog`.
2. Бэк **холдит** сумму на балансе (атомарно, с `idempotency_key`).
3. Вызов модели AI Tunnel.
4. Успех → холд превращается в списание по факту (для чата — по реальным токенам);
   ошибка → холд возвращается на баланс. Двойная запись в леджере.

### Интеграция AI Tunnel (OpenAI‑совместимый API)

`base_url = https://api.aitunnel.ru/v1`, заголовок `Authorization: Bearer <ключ>`.

- **Чат** (`gemini-3.1-flash-lite`, `gpt-5.5`): `POST /v1/chat/completions`.
- **Фото** (`gemini-3.1-flash-image`, `gemini-3-pro-image`): `POST /v1/images/generations`.
- **Видео** (`veo-3.1-lite`, `kling-v3.0-pro`): асинхронно —
  `POST /v1/videos` → опрос `polling_url` → готовый mp4 из `unsigned_urls[0]`.
- **Аудио→текст** (`voxtral-mini-transcribe`): `POST /v1/audio/transcriptions` (multipart).
